# 記憶醸造所 高忠実度HTMLプロトタイプ

## 内容

- `index.html`：参加者スマートフォン向け体験
- `venue.html`：会場スクリーン向け集合作品
- `styles.css`：デザインシステムとレスポンシブスタイル
- `app.js`：三段仕込みの画面遷移、Canvas描画、簡易Web Audio
- `venue.js`：会場作品のアニメーションと参加シミュレーション

## 起動方法

ローカルで簡易HTTPサーバーを起動してください。

```bash
python -m http.server 8080
```

その後、次を開きます。

- 個人体験: `http://localhost:8080/index.html`
- 会場作品: `http://localhost:8080/venue.html`

直接HTMLを開いても多くの機能は動きますが、ブラウザの音声制限や将来のAPI連携を考慮し、HTTPサーバー経由を推奨します。

## プロトタイプの意図

- SaaS風カードUIや紫系グラデーションを避ける
- 「水・米・麹・発酵・余韻」を抽象表現する
- 主要操作はスマートフォン下部に寄せる
- 初添ではフォームではなく作品そのものを操作する
- 踊りは入力工程ではなく、待つ体験として設計する
- 仲添では石川オープンデータ由来の土地候補を選ぶ
- 開栓画面はUIを抑え、音と動くビジュアルを主役にする

## Codex実装時の扱い

このディレクトリを承認済みデザインリファレンスとして扱い、以下を原則維持してください。

- 色、余白、文字サイズ、画面構成
- 三段仕込みの操作順
- 主要アニメーションの速度感
- 初添のCanvas操作
- 開栓画面の全面作品構成
- 会場作品と個人作品の役割分離

DOM構造をそのままReactへ移植する必要はありません。視覚と操作を維持したまま、適切なコンポーネント構造へ再設計してください。
