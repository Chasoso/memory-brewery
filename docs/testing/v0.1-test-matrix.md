## Screenshot baseline policy

Canvas rasterisation differs between Windows and Linux, so platform-specific baselines are intentional. Baselines use Chromium, DPR 1, `ja-JP`, `Asia/Tokyo`, light color scheme, reduced motion, E2E mode, fixed seed/clock, a 20 ms completed opening, and an isolated localStorage snapshot. Participant screenshots use 375×760; venue screenshots use 1280×720. The multiple-recipe venue state is produced from the two fixed synthetic sake recipes in stable recipe order.

Update only after checking the product spec and approved prototype, then review expected, actual, and diff images. CI never updates baselines. Windows: `npm run test:visual:update`; Linux baselines are updated only from a failed CI artifact after visual review. Do not raise diff thresholds, mask the Canvas, or update all baselines merely to make a test pass. Codex must not update baselines without that review.

`npm run test:e2e` is the required PR command. `npm run test:visual` selects screenshot tests; `npm run test:visual:update` is a local, explicit update command and is never used by CI.
