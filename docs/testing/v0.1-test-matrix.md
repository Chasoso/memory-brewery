## Baseline policy

Canvas rasterisation differs across Windows and Linux, so platform-specific baselines are intentional. Update only after checking the product spec and approved prototype, then review expected/actual/diff images. CI never updates baselines. Windows: `npm run test:visual:update`; Linux baselines are updated only from the failed CI artifact after visual review. Do not raise diff thresholds to hide changes.

`test:e2e` is the required PR command. `test:visual` selects screenshot tests; `test:visual:update` is a local, explicit update command and is not used by CI.
